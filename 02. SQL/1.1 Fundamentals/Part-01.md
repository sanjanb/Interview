## 1. Database & SQL Basics

* **SQL Definition:** Structured Query Language (SQL) is used to communicate with databases to retrieve, insert, update, and delete data. It relies on simple English-like commands.
* **DBMS & RDBMS:** A Database Management System (DBMS) is software that enables users to define, create, and manage databases (e.g., MySQL, MSSQL, PostgreSQL, Oracle). A *relational* database (RDBMS) specifically organizes data into tables consisting of rows (records) and columns (attributes).
* **CRUD Operations:** The fundamental operations for managing data:
* **C**reate $\rightarrow$ Insert new records.
* **R**ead $\rightarrow$ Retrieve data.
* **U**pdate $\rightarrow$ Modify existing data.
* **D**elete $\rightarrow$ Remove records.

## 2. Core SQL Statements & Clauses

* **SELECT:** Used to retrieve data. It uses the **WHERE** clause to narrow down data based on specific filtering criteria.
* **INSERT:** Used to add new rows of records into a specified table.
* **UPDATE:** Modifies existing entries inside a table based on conditions.
* **DELETE:** Erases specific records from a table using conditions.
* **JOINS:** Used to combine rows from two or more tables based on a related column between them. The video introduces four main types: *Inner Join, Left Join, Right Join,* and *Full Join*.

## 3. Structural Concepts & Rules

* **Data Types:** Defines what kind of data a column can store, such as `INTEGER`, `VARCHAR` (text), `DATE`, and `BOOLEAN`.
* **Constraints:** Rules enforced on columns to maintain data integrity:
* **Primary Key:** Ensures every value in that column is unique and not null.
* **Foreign Key:** Maintains referential integrity between two related tables.
* **Not Null:** Restricts a column from accepting null or empty values.
* **Unique:** Guarantees all values in a column are distinct.

* **Indexes:** Special database objects created to speed up data retrieval and improve query execution performance in large tables.

## 4. Data Aggregation & Advanced Retrieval

* **Aggregate Functions:** Functions that perform calculations on a set of values and return a single summary result (e.g., `SUM`, `AVG`, `COUNT`, `MIN`, `MAX`).
* **GROUP BY Clause:** Groups rows that have the same values into summary rows. It is heavily used alongside aggregate functions.
* **ORDER BY Clause:** Sorts the resulting dataset in either Ascending (`ASC`) or Descending (`DSC`) order based on specific columns or aggregated values.
* **Subqueries:** Also known as inner or nested queries. It is a query embedded inside another main query, usually acting as a complex filter or condition.

## 5. Advanced Database Architecture

* **Views:** Virtual tables generated by a query. They act like regular tables and are used to simplify complicated queries or hide/restrict access to sensitive columns.
* **Transactions & ACID Properties:** Sequences of SQL operations handled as a single unit of work to ensure they all succeed or all fail together. They maintain database consistency using the **ACID** properties: **A**tomic, **C**onsistent, **I**solated, and **D**urable.
* **Normalization:** The architectural process of splitting up data into multiple tables and creating relationships to reduce data redundancy and eliminate anomalies.
* **Stored Procedures:** Pre-compiled SQL code saved directly on the database server to optimize speed and enhance security.
* **Triggers:** Database objects that automatically execute a specified block of SQL code in response to events like data insertions, modifications, or deletions (`BEFORE`, `AFTER`, or `INSTEAD OF` an event).
